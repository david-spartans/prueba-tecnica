Requerimiento de Prueba Técnica - Frontend Engineer

Objetivo:

Desarrollar una aplicación Angular que consuma el endpoint RESTful proporcionado y muestre los datos en una lista.

Descripción del Proyecto:

  1. Buenas prácticas:
    • Utiliza las mejores prácticas al desarrollar esta prueba.
    • La prueba incluye la interfaz User para usarla en los servicios y donde sea necesario.
  2. Uso del Shell para generar los componentes:
    • El Shell se encuentra en la parte inferior derecha del navegador; aquí podrás ejecutar comandos ng para generar los componentes.
  3. Uso de Endpoints:
    • URL Base: https://jsonplaceholder.typicode.com
    • Endpoint para listar usuarios: ${BASE-URL}/users
    • Endpoint para detalles de usuario: ${BASE-URL}/users/${id}
    • Método: GET
    • Descripción: Devuelve una lista de usuarios y detalles de usuarios en formato JSON.
  4. Requisitos de Implementación:
    • Servicio:
      • Crea un servicio ApiService para realizar solicitudes HTTP y retornar los datos.
      • Genera un método para retornar los usuarios usando /users.
      • Genera un método para retornar el detalle del usuario usando /users/1.
    • Componente UserList:
      • Crea un componente UserListComponent que consuma el servicio y muestre la lista de usuarios.
      • Cada usuario debe mostrar al menos el nombre (name) y el correo (email).
      • Cada usuario tendrá un botón de "Ver" que redireccionará al detalle del usuario de acuerdo a su identificador (id).
      • Incorpora el componente FilterUser que retornará una cadena para generar una función de filtrado de usuarios (los requisitos de este componente se describen más adelante).
    • Componente UserDetailList:
      • Crea un componente UserDetailListComponent que consuma el servicio y muestre los detalles de usuario por ID.
      • Cada detalle de usuario debe mostrar al menos el ID del usuario (userId), el nombre (name), el usuario (username) y el teléfono (phone).
      • Este componente deberá tener la capacidad de regresar al listado de usuarios con un botón de "Regresar".
    • Componente FilterUser:
      • Crea un componente FilterUserComponent que deberá renderizarse dentro de UserList sobre el listado de usuarios.
      • Este componente deberá contener un input con un evento onSearch que retornará una cadena para filtrar los usuarios; la lógica de filtrado estará dentro del componente UserList.
    • Lógica de Filtrado de Usuarios en UserList:
      • Crea una función con la cadena que retornará filterUser mediante un EventEmitter.
      • Esta función deberá filtrar los usuarios a mostrar en el render.
      • La lógica de esta función es a tu consideración para resolver esta problemática.
  5. Estilos:
    • El proyecto ya cuenta con los estilos de Bootstrap previamente cargados para que puedas hacer uso de esta librería.
  6. Instrucciones para la Entrega:
    • Tiempo: Tendrás 1 hora con 30 minutos para completar la prueba.
    • Prueba: Asegúrate de que el proyecto funcione correctamente en el entorno CodeInterview.io.
    • Restricciones: Queda prohibido el uso de Inteligencia Artificial (IA) para resolver esta prueba (incluye chats, extensiones en VSCode y el uso de copilots).
  7. Criterios de Evaluación:
    • Correcta implementación del servicio y los componentes.
    • Buenas prácticas al codificar.
    • Claridad y calidad del código.
